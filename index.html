October 27th, 2020

Bugs fixed:

Unit text height still using feet instead of inches if location was set to bottom of unit

Units could be created without Customers or Tags

Customer and Unit Lists no longer are too wide

Features:

Implemented Add and Remove buttons. Add will place the unit into the list and not the truck. Remove appears on editing.

Unit's in list can now drag and drop to the truck. If not in truck, unit's appear in red.

Unit height and tag font is bigger

Print now displays the height counters


October 26th, 2020

Bugs fixed:

Fixed silly mistake using feet' instead of inches" Whoopsie!

Rotating text always placed location on bottom rather than beside if it could fit

Comment tool now doesn't snap to grid or units allowing to be placed anywhere

Location not going to blank after unselecting unit

Tabbing to location now skips width and height sliders

Edit button sometimes needed to be clicked twice

Rare bug making it possible to change a customers drop and not the associated units drop

Preview would be stretched if location was on bottom

Updating unit with no location to a location doesn't place location beside tag

Features:

Implemented previous version link. This is a safety net in case a new version has a show stopping bug you can opt to use the older version

Tag and Location information now only appears on edit instead of on selecting unit

Bugs fixed:
